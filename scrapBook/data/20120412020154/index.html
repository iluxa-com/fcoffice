Google雲數據庫Google Cloud SQL使用手記

作者: calvinwu   分類：免費空間   出處：本站原創       大 | 中 | 小

分享到： 新浪微博 騰訊微博 QQ空間 開心網 谷歌Buzz 百度搜藏 QQ收藏


傳統的應用大多使用關系型數據庫作數據存儲，但由於關系型數據庫對做系統擴展時通常需要進行大量的修改，所以這類系統起初都會靠升級系統硬件來增加性能，但如果硬件升級愈高，性價比會愈低，所以對這種系統做擴展很困難。而 Google App Engine 的 Datastore 是 NoSQL Database，這種數據庫擴展系統是比較簡單的，比如可以買 10 臺中級機器去組成一個高級的系統，如果不夠可以再添加機器，理論上系統可以無限伸延。NoSQL Database 是沒有 Join Table 之類的語法的，你只能靠應用程序層面來實現 Join Table，當然這會從某種程度上增加程序開發的難度。
對於很多熟悉關系型數據庫，又想嘗試Google App Engine的開發者來說，Datastore成了他們的夢魘，而且要把現有應用移植到 Google App Engine 的最大問題也是 Datastore，很多人因此卻步。
但一切都會因為Google Cloud SQL的推出而有所好轉，上次介紹了GOOGLE CLOUD SQL 主要特性和功能限制，並介紹了申請步驟，經過差不多三天的等待後，申請通過，之後我嘗試了建立Google Cloud SQL數據庫，並往其中導入數據，操作不難，感覺還不錯！
